# 사용자 회로도(수기 설계) 검토 기록 — 2026-07-12

`/discrete-potentiostat/firmware` 가 타깃하는 실제 보드는 사용자가 KiCad 로 직접 그린
회로도다 (이 저장소의 `hardware/kicad` RevC 와는 별개 설계 — 토폴로지는 동일).
이 문서는 그 회로도에 대한 리뷰 결론과 확정 파라미터의 기록이다.

## 확정 핀맵 (펌웨어 `app.overlay` 와 1:1)

| 기능 | 핀 | 비고 |
|---|---|---|
| TIA 출력 (SAADC 차동 +) | P0.02/AIN0 | |
| VREF 1.024 V (차동 −) | P0.03/AIN1 | REF35102 |
| VBAT 모니터 | P0.04/AIN2 | 1M:1M 분압 (`Vbat` 넷 = 스위치 앞) |
| Qi 패드 감지 (`qi`) | P0.05/AIN3 | MCP73832_VDD(≈5 V) 1M:1M 분압 |
| AVDD 급전 (AFE 전원) | **P0.26** (QFN 38) | high-drive. R7 삭제로 DVDD 와 완전 분리 |
| 충전 상태 (`chg`) | P0.08 | MCP73832 STAT, 100k→DVDD 풀업, LOW=충전중 |
| LED (옵션) | P0.06 | 기본 미실장 |

## 전원 토폴로지 (치명 오류 2회 수정 후 확정)

```
Qi 5V(BQ51013B OUT) ── MCP73832_VDD ── U7 VDD + C19/C27 + R15(Charge 분압)
U7 VBAT ── Vbat ── 배터리+ + C28 + 스위치 공통 + R13(VBAT 분압)   ← 충전: 스위치 무관
스위치 출력 ── 3V7 ── LDO IN/EN                                   ← 기기: 스위치 제어
AVDD ← P0.26 단독 급전 (측정 시에만 ON → 분극 방지)              ← AFE: 펌웨어 제어
```

수정 이력: ① 초판 — U7 VBAT 미연결(충전 불가) + 스위치 양단이 같은 넷(무효) +
Charge 가 ADC 없는 P0.06. ② 2판 — BQ OUT↔스위치출력 라벨 맞바뀜(5V 배터리 직결
위험) + R7 잔존. ③ 3판에서 전부 해소 확인.

## 확정 부품 값

| 항목 | 값 | 근거 |
|---|---|---|
| Qi 코일 | Würth **760308101214** (26 µH ±10 %, Ø19, DCR 0.52 Ω) | 사용자 선정 |
| C13 (직렬 공진) | **82 nF** C0G 50 V **1206** | Ls'(패드 위 +15 %) 기준 fs≈100 kHz. 68/100 nF 예비로 실측 튜닝 |
| C24 (병렬 공진) | **1 nF** C0G 50 V 0402 | fd≈1 MHz (fd 는 허용폭 넓음) |
| R5 (PROG) | **49.9 k** → 20 mA | LIR2032 40 mAh 의 0.5C (25 k=40 mA=1C 는 과함) |
| R6 / C31 (제어루프) | **1 M / 10 nF** (τ=10 ms) | 10 M/100 nF(τ=1 s)에서 CA 응답 개선 |

| R9 (TIA RF) | 2 M ∥ C32 100 pF | 펌웨어 기본 `rf`=2 M 일치. 레인지 약 −0.5~+1.1 µA |

## 수동소자 사이즈/정격 규칙 (0402 통일 불가 항목)

AC1/AC2/RECT 에 닿는 캡은 출력 5 V 와 무관하게 공진 배율·부하급변으로 수십 V 를 본다.

| Ref | 사이즈 | 사양 | 이유 |
|---|---|---|---|
| C13 | **1206** | 82 nF C0G 50 V | 0402 에 C0G 50 V 없음. X7R 대체 시 공진 표류(충전 간헐) — 시제품도 불가 |
| C22 (RECT) | **0805** | 10 µF X5R 25 V | RECT 평시 5~7 V + 스파이크 15 V+. 0402(6.3 V)는 파손 위험 — 시제품도 불가 |
| C18, C20 (CLAMP) | 0603 | 470 nF X7R 25 V | 클램프 동작 시 AC 피크 직격 |
| C19, C27, C28 | 0603 | 10µ/4.7µ X5R **10 V** | DC 바이어스 감쇄 — 0402 는 실효 30~40 % (충전기 발진 위험) |
| C11 (DVDD) | 0603 | 4.7 µF X5R 6.3 V↑ | Nordic 요구 4.7 µF 실효 확보 |
| C17/C21/C25/C26 | 0402 유지 | 10n/15n **50 V** X7R | AC 소용량은 50 V 품 지정으로 해결 |
| 나머지 전부 | 0402 | — | 저항 전체 포함 (µA~mA 라 발열 무관) |

참고: MCP73832 요구 최소는 4.7 µF — 초기 47 µF 는 과사양이었음. C27/C28 = 4.7 µF 0603 10 V 로 확정.

## 앱 배터리 표시 가이드

- status `vbat`[mV] → 잔량 룩업: 4200=100 % / 4000≈85 / 3900≈70 / 3800≈55 / 3700≈35 /
  3600≈20 / 3450≈10 / 3300=경고(`lb:1`) / 3000=컷오프. 충전 중엔 % 대신 "충전중" 표시.
- 충전 완료 판정: **`qi:1 && chg:0`** (패드 위 + STAT 해제 = 종지전류 도달).
- 3상태: 패드 밖(`qi:0`) / 충전중(`qi:1,chg:1`) / 완충(`qi:1,chg:0`).
