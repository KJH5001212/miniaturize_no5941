# 펌웨어 포팅 계획 (기존 `firmware/` → 디스크리트 AFE)

측정 방식이 근본적으로 다르다: 기존은 **AD5941이 측정하고 nRF는 SPI로 결과를 받아오는 구조**,
새 설계는 **nRF 자신이 ADC(SAADC)가 되어 직접 측정하는 구조**다.
따라서 측정 코어(아래 "신규 작성")는 포팅이 아니라 **완전 신규 코드**다 — AD5941 관련
코드(SPI, 시퀀서, 레지스터 제어)는 한 줄도 남지 않는다.

다만 기존 펌웨어에서 측정 코어는 "샘플 하나(전류, 타임스탬프)를 상위로 넘기는" 인터페이스로
앱 계층과 분리되어 있으므로, BLE/버퍼링/런 모드/프로토콜 계층은 수정 없이 재사용된다.

## 그대로 재사용

- NUS 기반 JSON 프로토콜 (`start`/`stop`/`config`/`ack`/`status`)
- `{"d":[[seq,t_ms,current_nA,range],...]}` 스트리밍 포맷 — `range` 필드는 SAADC 게인 인덱스로 의미만 변경
- 무손실 링버퍼 + seq/ACK 재전송
- 전력 인지형 BLE 커넥션 인터벌 (측정 중 fast / 대기 slow)
- 런 모드: 연속 / 타이머 / 사이클
- Android 앱 전체 (프로토콜 동일하므로 무수정)

## 제거

- `src/ad5940lib/` (벤더 드라이버) 및 SPI 버스 설정
- AD5941 초기화/시퀀서/LPTIA 오토레인지 코드

## 신규 작성

### 1. AFE 제어 (GPIO)
- `AFE_PWR` GPIO (고드라이브): 측정 시작 시 HIGH → REF/OPA2391 급전,
  안정화 대기 ~50 ms 후 샘플링 시작. 정지 시 LOW (전극 분극 방지 + 소모 0).

### 2. SAADC 측정 엔진
- 차동 채널: AIN+ = TIA_OUT, AIN− = 1.024 V 노드
- 타이머(RTC) 트리거 주기 샘플링, 하드웨어 오버샘플링 최대 활용 + 필요 시 소프트웨어 평균
- 변환: `current_nA = (code × LSB_nV[gain]) / RF_MΩ` (캘리브레이션 계수 적용)
- acquisition time 40 µs (1 kΩ 소스 임피던스 대응)

### 3. 게인 오토레인지 (AD5941 오토레인지 대체)
- 게인 테이블: {1/2, 1, 4} → range 인덱스 {0, 1, 2}
- |ΔV| > 85% FS → 게인 다운 / < 20% FS → 게인 업 (히스테리시스)
- 전환 직후 첫 샘플 폐기

### 3.5 RF/오버샘플링은 하드코딩 금지 — 런타임 설정값

RF와 CF는 보드에서 언제든 교체될 수 있는 부품이므로, 펌웨어가 RF에 의존하는 지점을
전부 설정값으로 뺀다:

- `rf_ohm` (전류 환산 `I = ΔV/RF` 및 게인 전환 임계값 계산에 사용)
- `oversample` (RF가 클수록 신호가 커져 낮춰도 됨: 1 MΩ→64×, 5.1 MΩ→16×, 10 MΩ→4× 수준)

둘 다 NVM(Settings) 저장 + BLE `config` 명령으로 변경 가능하게 한다.
→ RF 교체 절차 = ① 새 RF/CF 납땜 ② BLE로 `rf_ohm` 전송 ③ 더미셀 캘리브레이션 1회.
   펌웨어 재빌드/리플래시 불필요.

### 4. 캘리브레이션
- 더미셀(기지 저항 1.00 MΩ, 이론 전류 0.512 µA)로 게인별 스케일 보정
- 셀 오픈 상태에서 오프셋(제로 전류) 측정
- 계수는 NVM(Settings subsystem) 저장

### 5. 배터리 관리 (신규 — LIR2032는 맨셀)
- VBAT 분압 채널 주기 측정, `status` JSON에 전압 포함
- < 3.3 V: 앱에 low-batt 경고 / 측정 시작 거부
- < 3.0 V: 데이터 플러시 후 System OFF (과방전 보호)
- 충전 중(선택: BQ51050B CHG 핀 → GPIO) 측정 시작 시 노이즈 경고

## 보드 오버레이

- SPI 노드 삭제, `zephyr,user`에 SAADC 채널(차동 2입력 + VBAT) / AFE_PWR / CHG GPIO 정의
- 모듈 변경 시(MDBT42Q → ANNA-B112 등) 핀맵만 교체

## 검증 순서

1. 더미셀(저항+커패시터)로 스텝 응답 — TIA 안정성(발진 여부), CF 값 확정
2. 기지 저항 3점(100 k / 1 M / 10 M)으로 게인별 선형성·오토레인지 전환 확인
3. 페로시안화칼륨 등 표준 산화환원계로 실제 CA 곡선 vs AD5941 보드 교차 비교
4. 충전 중 노이즈 플로어 측정 → "충전 중 측정 금지" 정책 확정
